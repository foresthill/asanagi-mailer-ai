# Security Policy

Asanagi はメールという機微なデータを扱うため、セキュリティを最優先します。

## 脆弱性の報告

脆弱性を見つけた場合は、**公開Issueを立てず**、リポジトリオーナーに非公開で連絡してください
（GitHub の Security advisories、または `g1989n@gmail.com`）。

- 再現手順・影響範囲・想定される修正案を添えていただけると助かります。
- 対応方針をできるだけ早く返信します。

## 取り扱い上の原則

- **資格情報をコミットしない**: APIキー・OAuthトークン・メールパスワードは `.env.local`（gitignore済み）にのみ置く。`.env.example` はプレースホルダのみ。
- **メール本文・学習データは端末内で完結**できる設計を維持（Phase 2 のローカルファースト／PIIマスキング方針）。
- 外部AIプロバイダにメールを送る経路では、将来的にマスキング（PII除去）を挟めるようにする（[docs/00-overview.md](docs/00-overview.md) のフェーズ計画を参照）。

## 対応バージョン

Phase 1 MVP の段階です。セキュリティ修正は `main` の最新に対して行います。
