# Contributing to Asanagi（朝凪）

Asanagi への貢献に興味を持っていただきありがとうございます 🌊

> ステータス: **Phase 1 MVP（活発に開発中）**。設計は [`docs/`](docs/) を参照してください。API・UI・データモデルは変わり得ます。

## 開発環境

```bash
npm install
npm run dev        # http://localhost:3000（別プロジェクトのSWと衝突する場合は PORT=3100）
npm run build      # 型チェック込みの本番ビルド
npm run lint       # ESLint
```

APIキーなしでもモック受信箱で動作します。実接続は [`.env.example`](.env.example) を参照。

## 設計の前提（重要）

- **3つの差し替え可能なシーム**を壊さないこと: `EmailProvider`（メール）・`resolveModel()`（AI）・`store`（永続化）。機能コードは具体実装ではなくインターフェースに依存させる。
- **AIは勝手に上書きしない**: 提案 → ユーザーが採用/却下、を徹底（特に文章添削）。
- **プライバシー優先**: メール本文・学習データは端末内で完結できる設計を崩さない（Phase 2 のローカルファースト方針）。
- 詳細は [docs/00-overview.md](docs/00-overview.md) を最初に読んでください。

## プルリクエスト

1. `feature/<topic>` ブランチを切る（`main` への直接 push は不可）。
2. `npm run lint` と `npm run build` が通ること。
3. UIを変えた場合はスクリーンショットを添付。
4. 変更の「なぜ」をPR説明に書く。

## コミット

- 1コミット＝1つの意味のある変更。説明的なメッセージで（`fix`/`update` のみは避ける）。

## サポートについて

これは個人プロジェクトで、**現状有姿（as-is）／能動的サポートなし**です。PR は歓迎しますが、レビュー・マージは保証しません。**気軽にフォークして自分の手で進める**のがおすすめのスタンスです。
