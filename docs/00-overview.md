# Asanagi（朝凪）設計書 — 全体概要

最終更新: 2026-06-03

## 1. ビジョン

AIネイティブ・AIファーストなメールクライアント。メールを「読む・書く・さばく」すべての操作にAIが介在し、**人間は判断と意思決定に集中**できる状態をつくる。

中核となる4つの体験:

1. **AIが返信を提案し、会話で一箇所ずつ添削できる**（→ [01](01-incremental-reply-editing.md)）
2. **AIがメールの重要度を判定し、ユーザーごとに学習する**（→ [02](02-importance-triage-learning.md)）
3. **さばいたメールは消し込まれ、受信箱が常にきれいになる**（アーカイブ/ゴミ箱）
4. **即時送信 / スケジュール送信**

## 2. フェーズ計画

| フェーズ | 目的 | スコープ | 状態 |
|----------|------|----------|------|
| **Phase 1** | コア体験の確立（Web） | Gmail API接続・AI返信＋添削・重要度判定＋学習・送信/予約・アーカイブ/ゴミ箱 | MVP実装済み（モック）。本書で添削UXと学習機構を再設計 |
| **Phase 2** | 実運用（会社メール）＋ネイティブ＋整理 | IMAP/SMTP対応・**Macネイティブアプリ**・**スレッド表示**・**インボックス整理（バンドル/ビュー）**・PIIマスキング・ローカルLLM/ローカル学習・承認ワークフロー基盤 | 設計 |
| **Phase 3** | 組織利用 | 承認ワークフロー本格化・チーム共有ナレッジ・高度な学習（分類器/埋め込みRAG） | 構想 |

> 補足（ユーザー方針）: 本命は会社メール（IMAP/SMTP）。Gmailは8万通超あるため**接続以降の新着のみ同期**する（過去は backfill しない）。詳細は [03](03-providers-sync-native.md)。

## 3. アーキテクチャ原則

### 3.1 差し替え可能な3つのシーム（seam）

機能コード（UI / APIルート）は、以下のインターフェースにのみ依存し、具体実装に直接依存しない。

```
┌─────────────────────────────────────────────┐
│            UI (React) / API Routes           │
└───────────────┬──────────────┬──────────────┘
                │              │
       EmailProvider      AI Model           Store
   (gmail/imap/mock)  (claude/openrouter/  (json→sqlite→
                       gateway/local)        local-first)
```

| シーム | 抽象化 | 現状 | 将来 |
|--------|--------|------|------|
| メール | `EmailProvider` (`src/lib/email/provider.ts`) | Gmail / IMAP / Mock | アカウント複数・ローカル同期エンジン |
| AI | `resolveModel()` (`src/lib/ai/model.ts`) | Claude/OpenAI/OpenRouter/Gateway | ローカルLLM (MLX/Ollama) |
| 永続化 | `store.ts` | ローカルJSON | SQLite → ローカルファースト同期 |

### 3.2 ローカルファースト原則（Phase 2以降）

会社メール対応とMacネイティブ化に向け、**ローカルSQLiteを正本（source of truth）**とし、サーバ（Gmail/IMAP）とは同期エンジンで突き合わせる。メール本文・学習データ・埋め込みは**端末外に出さないことを既定**とする（プライバシー＝企業導入の前提）。

### 3.3 AIの透明性原則

- AIの判定・提案には**必ず理由（reason）を添える**（重要度の根拠、添削の意図）。
- AIは**勝手に上書きしない**。提案 → ユーザーが採用/却下、を徹底する（特に文章添削）。
- 学習は**ユーザーが見て・編集できる**（ブラックボックスにしない）。

## 5. 用語

- **添削（refine）**: 既存下書きへの部分的修正提案。
- **シグナル（signal）**: 重要度判定に使う学習済みの手がかり（送信者/ドメイン/キーワード等）。
- **嗜好プロファイル（preference profile）**: フィードバック履歴をAIが要約した「この人が重要視するもの」の自然言語記述。
- **新着のみ同期（forward-only sync）**: 接続時点を基準に、以降の変更だけ取り込む方式。
